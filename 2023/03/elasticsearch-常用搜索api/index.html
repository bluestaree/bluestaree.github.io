Elasticsearch 常用搜索API

  2023-03-27   elasticsearch   Elasticsearch  评论 字数统计: 1460字 阅读时长: 3分

Elasticsearch 常用搜索API

1.查看es中有哪些索引

GET /_cat/indices?v

es 中会默认存在一个名为.kibana的索引

表头的含义

health green(集群完整) yellow(单点正常、集群不完整) red(单点不正常)
status 索引打开、关闭状态
index 索引名
uuid 索引统一编号
pri 主分片数量
rep 副本数量
docs.count 可用文档数量
docs.deleted 文档被删了多少（逻辑删除）
store.size 整体（主分片和副分片）占空间大小
pri.store.size 主分片占空间大小
cluster 整个elasticsearch 默认就是集群状态，整个集群是一份完整、互备的数据。
node 集群中的一个节点，一般只一个进程就是一个node
shard 分片，即使是一个节点中的数据也会通过hash算法，分成多个片存放，默认是5片。
index 相当于rdbms的database, 对于用户来说是一个逻辑数据库，虽然物理上会被分多个shard存放，也可能存放在多个node中。
type 类似于rdbms的table，但是与其说像table，其实更像面向对象中的class , 同一Json的格式的数据集合。
Document(json) 类似于rdbms的 row、面向对象里的object
field 相当于字段、属性

2.增加一个索引(库）

PUT /movie_index

3.删除一个索引

DELETE /movie_index

4.新增文档

// 格式 /索引/类型/id 
PUT /movie_index/_doc/1
{
  "id": 1,
  "name": "operation red sea",
  "doubanScore": 8.5,
  "actorList": [
    {
      "id": 1,
      "name": "zhang yi"
    },
    {
      "id": 2,
      "name": "hai qing"
    },
    {
      "id": 3,
      "name": "zhang han yu"
    }
  ]
}

PUT /movie_index/_doc/2
{
  "id": 2,
  "name": "operation meigong river",
  "doubanScore": 8,
  "actorList": [
    {
      "id": 3,
      "name": "zhang han yu"
    }
  ]
}

// 也可用POST请求，如果不指定id 将会随机生成
POST /movie_index/_doc/3
{
  "id": 3,
  "name": "incident red sea",
  "doubanScore": 5,
  "actorList": [
    {
      "id": 4,
      "name": "zhang chen"
    }
  ]
}

如果之前没建过index，es 会自动创建。

5.直接用id查找

GET /movie_index/_doc/1

6.修改—整体替换

和新增没有区别, 注意返回信息中得_version值

PUT /movie_index/_doc/3
{
  "id": "3",
  "name": "incident red sea",
  "doubanScore": "5.0",
  "actorList": [
    {
      "id": "1",
      "name": "zhang chen"
    }
  ]
}

// 返回信息
{
  "_index" : "movie_index",
  "_type" : "_doc",
  "_id" : "3",
  "_version" : 2,

    "total" : 2,
    "successful" : 1,
    "failed" : 0
  },

  "_primary_term" : 1

    "doubanScore":"7.0"

10.按条件查询(全部)

GET /movie_index/_search
{
  "query":{
    "match_all": {}
  }
}

11.多条件查询

GET /movie_index/_search
{
  "query": {
    "multi_match": {
      "query": "red",
      "fields": ["name","actorList.name"]
    }
  }
}

12.按分词查询

GET /movie_index/_search
{
  "query":{
    "match": {"name":"red"}
  }
}

13.按分词子属性查询

GET /movie_index/_search
{
  "query":{
    "match": {"actorList.name":"zhang"}
  }
}

14.过滤–查询后过滤

类似having

GET /movie_index/_search
{
    "query":{
      "match": {"name":"red"}
    },
    "post_filter":{
      "term": {
        "actorList.id": 3
      }
    }
}

15.过滤–查询前过滤（推荐）

类似where

GET /movie_index/_search
{ 
    "query":{
        "bool":{
          "filter":[ {"term": {  "actorList.id": "1"  }},
                     {"term": {  "actorList.id": "3"  }}
           ], 
           "must":{"match":{"name":"red"}}
         }
    }
}

16.过滤–按范围过滤

GET /movie_index/_search
{
   "query": {
     "bool": {
       "filter": {
         "range": {
            "doubanScore": {"gte": 8}
         }
       }
     }
   }
}

关于范围操作符

操作符 说明
gt 大于
lt 小于
gte 大于等于
lte 小于等于

17.排序

GET /movie_index/_search
{
  "query":{
    "match": {"name":"red sea"}
  }
  , "sort": [
    {
      "doubanScore": {
        "order": "desc"
      }
    }
  ]
}

18.分页查询

GET /movie_index/_search
{	
  "query": { "match_all": {} },
  "from": 1,
  "size": 1
}

19.指定查询的字段

GET /movie_index/_search
{
  "query": { "match_all": {} },
  "_source": ["name", "doubanScore"]
}

20.高亮

GET /movie_index/_search
{
    "query":{
      "match": {"name":"red sea"}
    },
    "highlight": {
      "fields": {"name":{} }
    }
}

21.match_phrase

按短语查询

注意:

1,match_phrase 会分词

2,被检索得字段必须包含match_phrase中所有词项并且顺序相同

3,被检索字段包含得match_phrase中得词项 之间 不能有其他词项

GET /movie_index/_search
{
    "query":{
    	"match_phrase": {"name":"operation red"}
    }
}

以下查询性能较低，不推荐使用

22.fuzzy 模糊查询

校正匹配分词，当一个单词都无法准确匹配，es通过一种算法对非常接近的单词也给与一定的评分，能够查询出来，但是消耗更多的性能。

混淆字符(box -> fox) 缺少字符(black -> lack)

多出字符(sic -> sick) 颠倒顺序(act -> cat)

GET /movie_index/_search
{
    "query":{
      "fuzzy": {"name":"rad"}
    }
}

23.prefix 前缀查询

GET /movie_index/_search
{	
  "query": { 
    "prefix": {
      "name": {
        "value": "op"
      }
    }
  }
}

24.wildcard 通配符

GET /movie_index/_search
{
  "query": {
    "wildcard": {
      "name": {
        "value": "oper*tion"
      }
    }
  }
}

25.regexp 正则

GET /movie_index/_search
{
  "query": {
    "regexp": {
      "name": {
        "value": "[\\s\\S]*incident[\\s\\S]*"
      }
    }
  }
}
