查找算法

  2020-03-23   数据结构与算法   算法  评论 字数统计: 781字 阅读时长: 2分

常用的查找算法：

1）顺序(线性)查找

- 有序与无序数组均可。逐一比对，发现相同值，就返回

2）二分查找/折半查找

前提：必须为有序数组

二分查找的思路分析

1.首先确定该数组的中间的下标

​ mid= (left + right)/ 2

2.然后让需要查找的数findVal和arr[mid]比较

  1. findVal>arr[mid],说明你要查找的数在mid的右边，因此需要递归的向右查找
  2. findVal< arr[mid],说明你要查找的数在mid的左边，因此需要递归的向左查找
  3. findVal== arr[mid]说明找到，就返回

什么时候我们需要结束递归.

  • 找到就结束递归

  • 递归完整个数组，仍然没有找到findVal，也需要结束递归 当left> right就需要退出

3）插值查找

前提：必须为有序数组

  1. 插值查找算法类似于二分查找，不同的是插值查找每次从自适应mid处开始查找。

  2. 下面是折半查找中的求mid索引的公式,其中low表示左边索引left,high表示右边索引right.key就是前面我们讲的

findVal

求中值公式

int mid = low + (high - low) * (key - arr[low])/ (arr[high] - arr[low]) ;

4）斐波那契查找 (黄金分割法)

前提：必须为有序数组

斐波那契查找原理与前两种相似，仅仅改变了中间结点 (mid) 的位置，mid不再是中间或插值得到，而是位于黄金

分割点附近，即mid=low+F(k-1)-1(F代表斐波那契数列)，如下图所示

对F(k-1)-1的理解:

  1. 由斐波那契数列F[k]=F[k-1]+F[k-2]的性质，可以得到 (F[k]-1) = (F[k-1]-1) + (F[k-2]-1) +1。该式说明:只要顺序

表的长度为F[k]-1,则可以将该表分成长度为F[k-1]-1和F[k-2]-1的两段，即如上图所示。从而中间位置为

mid=low+F(k-1)-1

  1. 类似的， 每一子段也可以用相同的方式分割

  2. 需要注意的是需要排序的顺序表长度n,不一定刚好等于F[k]-1, 所以需要将原来的顺序表长度n增加至F[k]-1后,才

能使用斐波那契查找算法。这里的k值只要能使得F[k]-1恰好大于或等于n即可。
