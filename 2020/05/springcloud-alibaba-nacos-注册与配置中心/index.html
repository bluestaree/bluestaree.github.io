SpringCloud Alibaba-Nacos 注册与配置中心

  2020-05-31   笔记   SpringCloud Nacos  评论 字数统计: 1316字 阅读时长: 3分

由于Spring Cloud Eureka已经宣布停止开源计划，不再继续维护，这怎么行，还能愉快的使用吗。 今天在空闲之余了解了下另外一个功能更加强大的注册中心，那就是SpringCloud Alibaba的Nacos

SpringCloud Alibaba

Spring Cloud Alibaba致力于提供微服务开发的一站式解决方案。此项目包含开发分布式应用微服务的必需组件,方

便开发者通过 Spring Cloud编程模型轻松使用这些组件来开发分布式应用服务。

依托 Spring Cloud Alibaba,您只需要添加一些注解和少量配置,就可以将 Spring Cloud应用接入阿里微服务解决方

案,通过阿里中间件来迅速搭建分布式应用系统。

GitHub地址：https://github.com/alibaba/spring-cloud-alibaba

本文主要介绍Spring Cloud Alibaba项目下的Nacos组件基本使用。

Nacos初体验

使用Nacos作为注册中心

1.首先引入SpringCloud Alibaba依赖 ,但是需要注意版本选择

  • 1.5.x 版本适用于 Spring Boot 1.5.x
  • 2.0.x 版本适用于 Spring Boot 2.0.x
  • 2.1.x 版本适用于 Spring Boot 2.1.x
  • 2.2.x 版本适用于 Spring Boot 2.2.x

pom文件配置：

<dependencyManagement>
    <dependencies>
        <dependency>
            <groupId>com.alibaba.cloud</groupId>
            <artifactId>spring-cloud-alibaba-dependencies</artifactId>
            <version>2.2.0.RELEASE</version>
            <type>pom</type>
            <scope>import</scope>
        </dependency>
    </dependencies>
</dependencyManagement>

2.引入Nacos注册中心依赖

pom文件配置：

 <dependency>
     <groupId>com.alibaba.cloud</groupId>
     <artifactId>spring-cloud-starter-alibaba-nacos-discovery</artifactId>
 </dependency>

配置文件的配置

server:
  port: 8080
spring:
  application:
    name: Hello-Nacos
  cloud:	
    nacos:	//这里配置nacos注册中心的地址
      discovery:
        server-addr: 127.0.0.1:8848

启动类

@EnableDiscoveryClient
@SpringBootApplication
public class WebApplication {
    public static void main(String[] args) {
        SpringApplication.run(WebApplication.class, args);
    }
}

获取Nacos的服务端 : https://github.com/alibaba/nacos/releases

下载后启动服务端，打开浏览器访问http://127.0.0.1:8848/nacos

默认账号密码均为 : nacos

启动测试服务后，查看nacos服务管理中心，可以看到该服务已经成功注册

之后就可以结合spring cloud open-feign进行远程服务调用啦，这里不再演示

使用Nacos作为配置中心

首先导入nacos配置中心的依赖

<dependency>
     <groupId>com.alibaba.cloud</groupId>
     <artifactId>spring-cloud-starter-alibaba-nacos-config</artifactId>
 </dependency>

新增bootstrap.properties配置文件,添加已下配置信息

spring.application.name=hello-nacos		//项目名
spring.cloud.nacos.config.server-addr=127.0.0.1:8848	//nacos配置中心地址

在配置中心中新建配置文件,nacos配置中心默认会加载项目名.properties文件

Controller相关代码

@RestController
public class WebController {

    @Value("${mycustomer.name}")
    public String name;
    @Value("${mycustomer.age}")
    public int age;

    @GetMapping
    private Map test() {
        HashMap hashMap = new HashMap();
        hashMap.put("name", name);
        hashMap.put("age", age);
        return hashMap;
    }
}

结果

成功读取配置中心里配置信息

这里还可以配合使用**@RefreshScope**注解实现动态配置刷新

注意：如果配置中心和当前应用的配置文件中都配置了相同的项，会优先使用配置中心里的配置

实现多配置文件加载(如开发，生产，测试等环境…)

nacos配置中心的一些概念

  1. 命名空间,默认：public(保留空间)，默认新增的所有配置都存在public空间,

    要使用特定的命名空间，需要在bootstrap.properties配置文件中指定命名空间ID

    spring.cloud.nacos.config.namespace=23902197-c354-4216-94cf-8e5cb8537e26

    通过这一点，我们可以为每一个微服务创建一个命名空间，形成基于微服务间的隔离

  2. 配置集：所有配置文件的集合

  3. 配置集ID：相当于文件名

  4. 配置分组：默认所有配置集都属于DEFAULT_GROUP，我们可以在创建配置文件时，使用配置分组来区分不同的环境

    在bootstrap.properties配置文件 中指定分组

    spring.cloud.nacos.config.group=dev	//指定配置分组

bootstrap.properties配置文件中指定多配置文件加载

#多配置文件加载

spring.cloud.nacos.config.ext-config[0].data-id=datasource.yml	//配置ID

spring.cloud.nacos.config.ext-config[0].group=dev	//配置分组

spring.cloud.nacos.config.ext-config[0].refresh=true	//自动刷新

spring.cloud.nacos.config.ext-config[1].data-id=hello-nacos.properties

spring.cloud.nacos.config.ext-config[1].group=dev

spring.cloud.nacos.config.ext-config[1].refresh=true
