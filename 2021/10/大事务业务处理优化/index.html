大事务业务处理优化

  2021-10-14   笔记  评论 字数统计: 1147字 阅读时长: 3分

概述：起因是测试时遇到一个业务响应异常缓慢问题，排查后发现时mysql链接等待导致业务超时，定位到具体业务逻辑，初步判断应该是事务问题。这个业务内部逻辑繁多，而且存在远程调用这种耗时较高的行为，因为当时排期紧张，在实现完业务后，便没有考虑到这方面隐藏问题，在方法入口上加了**@Transactional**注解后草草了事。后续经过优化解决

业务伪代码

@Transactional
public int expireHouseholder(Householder householder){
	// 获取详情信息
	householderInfoService.getById(id);
	// 规则判断
	...
	...
	// 更新数据
    hmBaseMapper.updateById(householder);
    // 远程调用
    rpcMethod();
    // 后续操作
    otherDoing();
    return householder.getId();
}

问题：

  • 事务中包含与跟新数据无法的业务逻辑，查询、各种规则判断等
  • 事务中嵌套使用耗时服务（远程调用）

解决优化

TransactionTemplate

在不影响现有逻辑下，使用编程式事务 ： 通过 TransactionTemplate或者 TransactionManager 手动管理事务，将事务中包含与跟新数据无法的业务逻辑，查询、各种规则判断等提取到事务外中

@Autowired
private TransactionTemplate transactionTemplate;

public int expireHouseholder(Householder householder){
	// 获取详情信息
	householderInfoService.getById(id);
	// 规则判断
	...
	...
    // 开启事务
    transactionTemplate.execute((status) -> {  
      	// 更新数据
    	hmBaseMapper.updateById(householder);
    	...
    	...      
    }); 

    // 远程调用
    rpcMethod();
    // 后续操作
    otherDoing();
    return householder.getId();
}

TransactionSynchronizationManager

如果远程调用业务中，需要等待事务结束后操作，如发送系统消息等，需要明确等待事务提交完成后再执行（事务提交后，后续查询才能查出正确数据），可以结合使用TransactionSynchronizationManager

TransactionSynchronizationManager中提供了让我们对事务注册的方法：

public static void registerSynchronization(TransactionSynchronization synchronization)
    throws IllegalStateException {

    Assert.notNull(synchronization, "TransactionSynchronization must not be null");
    Set<TransactionSynchronization> synchs = synchronizations.get();
    if (synchs == null) {
        throw new IllegalStateException("Transaction synchronization is not active");
    }
    synchs.add(synchronization);
}

其中参数TransactionSynchronization 是一个接口，提供了一些钩子方法给我们， 可以根据需求实现对应的方法

public interface TransactionSynchronization extends Flushable {

   int STATUS_COMMITTED = 0;

   int STATUS_ROLLED_BACK = 1;

   int STATUS_UNKNOWN = 2;


    /**
    * 挂起时触发
    */
    default void suspend() {
    }

    /**
    * 挂起事务抛出异常的时候 会触发
    */
    default void resume() {
    }


   @Override
   default void flush() {
   }

    /**
    * 在事务提交之前触发
    */
   default void beforeCommit(boolean readOnly) {
   }

	/**
	 * 在事务完成之前触发
	 */
   default void beforeCompletion() {
   }

	/**
	 * 在事务提交之后触发
	 */
   default void afterCommit() {
   }

	/**
	 * 在事务完成之后触发
	 */
   default void afterCompletion(int status) {
   }

}

使用TransactionSynchronizationManager在事务提交之后操作

CallBackService

@Component
public class CallBackService {

    public void execute(final CallBackAction action) {

        // 事务是否处于活跃状态
        if (TransactionSynchronizationManager.isActualTransactionActive()) {
            TransactionSynchronizationManager
                    .registerSynchronization(new TransactionSynchronizationAdapter() {
                        // afterCompletion() 在事务完成之后触发
                        @Override
                        public void afterCompletion(int status) {
                            // 判断事务是否已完成
                            if (status == STATUS_COMMITTED){
                                // 事务提交后，开启新线程，处理后续业务
                                AsyncManager.me().execute(new TimerTask() {
                                    @Override
                                    public void run() {
                                    	// 业务处理
                                        action.callback();
                                    }
                                });
                            }else {
                                // 其他状态处理
                                
                            }
                        }
                    });
        } else {
            // 事务处于非活跃状态
            AsyncManager.me().execute(new TimerTask() {
                @Override
                public void run() {
                    action.callback();
                }
            });
        }

    }
}

CallBackAction

public interface CallBackAction {

    /**
     * 函数式接口
     */
    void callback();
}

使用

@Autowired
private TransactionTemplate transactionTemplate;

@Autowired
private CallBackService callBackService;

public int expireHouseholder(Householder householder){
	// 获取详情信息
	householderInfoService.getById(id);
	// 规则判断
	...
	...
    // 开启事务
    transactionTemplate.execute((status) -> {  
      	// 更新数据
    	hmBaseMapper.updateById(householder);
    	...
    	...
        // 在事务提交后执行
    	callBackService.execute(() -> {
            // 远程调用 --这里发生异常 不会影响以上事务结果
            rpcMethod();
    	});
    }); 
    // 后续操作 --这里发生异常 不会影响以上事务结果
    otherDoing();
    return householder.getId();
}

通过这种方式，可以将事务控制再较小范围内，确保 更新数据 事务的结果不会受到 远程调用，以及 后续操作 的影响。
